## Capabilities

### 1. Quality Assessment

Evaluates a question on 5 dimensions (each scored 0-1):

| Dimension | What It Measures |
|-----------|-----------------|
| **الوضوح (Clarity)** | Is the question stem clear and unambiguous? |
| **جودة الخيارات (Options Quality)** | Are distractors plausible? Is there only one correct answer? |
| **جودة التفسير (Explanation Quality)** | Does the explanation actually explain why? |
| **ملاءمة الصعوبة (Difficulty Fit)** | Is the difficulty label appropriate? |
| **القيمة التعليمية (Educational Value)** | Does this question actually test understanding? |

Returns: Overall score (0-100), strengths, weaknesses, suggestions, estimated difficulty, summary in Arabic.

**Prompt:** `buildQualityCheckPrompt()` in `prompts.ts`

### 2. Explanation Generation

Generates a detailed Arabic explanation for a question:

- Explains why the correct answer is correct
- Explains why each wrong answer is wrong
- Written in clear, student-friendly Arabic
- Tailored to the question type (analogy, completion, etc.)

**Prompt:** `buildExplanationPrompt()` in `prompts.ts`

### 3. Difficulty Estimation

Analyzes a question and estimates its difficulty:

- Returns: `easy`, `medium`, or `hard`
- Based on: complexity of concepts, subtlety of distractors, reading level

**Prompt:** `buildDifficultyPrompt()` in `prompts.ts`

### 4. Study Plan Generation

Creates personalized study plans:

- Analyzes recent performance data
- Identifies weak categories
- Generates a 7-day study schedule
- Provides motivational messages
- Recommends study vs exam mode
- Identifies learning style

**Prompt:** `buildStudyPlanPrompt()` in `study-plan-prompt.ts`

### 5. Chat Assistant (Studio)

Conversational AI for content managers:

- Answers questions about content quality
- Helps with question improvement suggestions
- Full conversation history persisted per session
- Streaming responses (token by token)

**Endpoint:** `src/app/api/studio/chat/stream/route.ts`

## Prompt Design

All prompts are in Arabic and designed to return structured JSON:

```typescript
// Example: Quality check prompt
export function buildQualityCheckPrompt(params: {
  stem: string;
  options: Record<string, string>;
  correctKey: string;
  explanation?: string;
  studyTip?: string;
  difficulty: string;
}): string {
  return `أنت خبير تقييم جودة أسئلة اختبار القدرات اللفظية.

قم بتقييم السؤال التالي وفق المقاييس التالية...

أعد النتيجة بصيغة JSON:
{
  "scores": { "clarity": 0-1, "options_quality": 0-1, ... },
  "overallScore": 0-1,
  "strengths": [...],
  "weaknesses": [...],
  "suggestions": [...],
  "estimatedDifficulty": "easy|medium|hard",
  "summary": "..."
}`;
}
```

## Fallback Strategy

If Gemini is unavailable (no API key, network error, rate limit):

1. **Quality check** → Falls back to heuristic scoring in `scoring.ts`
2. **Difficulty estimation** → Heuristic based on word count, option similarity
3. **Explanation generation** → Not available (no fallback for free-text generation)
4. **Study plan** → Falls back to heuristic plan based on statistics only

The heuristic scoring evaluates:
- Clarity: stem word count, option length variance
- Difficulty: word difficulty proxy, option similarity
- Quality: presence of explanation, valid options

## Processing Pipeline

```
processQuestion(questionId)
  │
  ├─► scoreQuestionWithAI()    ──► Gemini quality assessment
  ├─► generateAIExplanation()  ──► Gemini explanation
  └─► estimateDifficultyAI()  ──► Gemini difficulty
       │
       ▼
  Save results to Question model
  Log to AIProcessingLog table
```

## Batch Operations

| Action | Server Action | Description |
|--------|---------------|-------------|
| Process source | `triggerFullSourceProcessing` | Run all AI on all questions in a source |
| Batch difficulty | `batchEstimateDifficultyForSource` | Difficulty only for all questions in source |
| Batch explanations | `batchGenerateExplanationsForSource` | Explanations only for all questions in source |
| Process all | `triggerBatchProcessing` | Run AI on all unprocessed questions |

## Configuration

Settings in the Studio (Settings page):

| Setting | Effect |
|---------|--------|
| Auto Quality Check on Import | Runs quality assessment immediately after import |
| Auto Difficulty Estimation | Estimates difficulty during import |
| Auto Explanation Generation | Generates explanations during import |

## Environment Variables

```env
# Required for AI features
GOOGLE_API_KEY=your-gemini-api-key

# Optional: model selection
AI_MODEL=gemini-2.0-flash
```

Without `GOOGLE_API_KEY`, AI features gracefully degrade:
- Quality scores use heuristic fallback
- Study plans use statistics-based heuristic
- Chat and explanation generation are disabled
